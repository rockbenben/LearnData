# 看板娘 live2d-widget 维护说明

基于上游 [stevenjoezhang/live2d-widget](https://github.com/stevenjoezhang/live2d-widget),**已升级到 1.0.1**,本站自托管。上游 1.0 起用 TypeScript 重写,支持 Cubism 2 + Cubism 4/5,`waifu-tips.js` 是 ES Module,通过 `import()` 动态加载 `chunk/` 里的引擎代码(所以 **`chunk/` 必须一并部署**)。本站直接用上游 `dist/` 产物,不在仓库内构建。

---

## 1. 目录结构

```
live2d-widget/
├─ v20260619/            ← 代码(带版本号目录,见下方“缓存”说明)
│  ├─ autoload.js          入口:自动定位资源路径 + 调 initWidget
│  ├─ waifu-tips.js (+.map) 主逻辑(压缩产物)
│  ├─ chunk/               cubism2 / cubism5 引擎(动态 import,必需)
│  ├─ waifu.css            样式
│  ├─ waifu-tips.json      提示文案 + 触发规则
│  └─ live2d.min.js        Cubism 2 Core
├─ live2d_api/           ← 模型(~136MB,固定路径,永不随版本移动)
│  ├─ model_list.json      模型清单(index 见第 3 节)
│  └─ model/…              各模型资源 + textures.cache
├─ 手动修改说明.md         ← 本文件
└─ LICENSE / _headers
```

**只有 `templateBuild.html` 里这一行引用它**(看板娘只在生产构建注入,`pnpm docs:dev` 不显示):

```html
<script src="/live2d-widget/v20260619/autoload.js"></script>
```

## 3. 换默认看板娘

**当前默认 = `modelId 4` / `textureId 1` = `ShizukuTalk/shizuku-pajama`。**

`model_list.json` 索引对照:

| modelId | 模型 | 数组成员(用 textureId 选) |
|---|---|---|
| 0 | Potion-Maker/Pio | — |
| 1 | Potion-Maker/Tia | — |
| 2 | bilibili-live/22 | —(有多套换装) |
| 3 | bilibili-live/33 | —(有多套换装) |
| 4 | ShizukuTalk 组 | 0=shizuku-48,**1=shizuku-pajama** |
| 5 | HyperdimensionNeptunia 组 | 0=neptune_classic,2=neptune_santa(圣诞装)… 共 0~19 |
| 6 | KantaiCollection/murakumo | — |

**改默认两步**(都在 `v20260619/`):

1. `autoload.js` → `initWidget({ modelId: 4, … })` 把 `4` 改成目标 modelId。
2. **数组型模型**要指定成员、或单一模型要指定材质 → `waifu-tips.js` 搜 `(isNaN(d)||isNaN(r))&&(r=1)`,把 `1` 改成目标 textureId(上游原值是 `0`,本站补丁成 `1`;1.0 没把材质做成配置项,只能这样打补丁)。

> ⚠️ 默认值**只对新访客生效**。老访客浏览器 `localStorage` 里存了上次选的 `modelId`/`modelTexturesId`,会盖过默认。要看新默认效果,控制台跑:
> `localStorage.removeItem('modelId');localStorage.removeItem('modelTexturesId');location.reload()`

|---|---|---|
| `autoload.js` | 路径**自动定位**(`document.currentScript.src` 推出代码目录 + `../live2d_api/`);保留 `if (screen.width < 768) return`(手机端不加载);`logLevel:"warn"` | 上游写死 jsDelivr 路径、不限手机 |

## 5. 换装(切衣服)与 textures.cache

1.0 的换装是纯前端静态实现:

- **数组型模型**(modelId 4、5 等):点"切换模型/换装"在数组成员间切换,**不需要** `textures.cache`。
- **单一模型**(22、33、Pio…):换装需要该模型目录下有 `textures.cache`(列出可换的贴图组)。

本站已从 fghrsh 拉取了**真实存在**的 7 个:`bilibili-live/22`、`bilibili-live/33`、`Potion-Maker/Pio`、`Potion-Maker/Tia`、`KantaiCollection/murakumo`、`ShizukuTalk/shizuku-48`、`ShizukuTalk/shizuku-pajama`。

> ⚠️ **`textures.cache` 被全局 gitignore 的 `*.cache` 规则忽略**,必须 `git add -f` 才能进库,否则线上(GitHub Actions 重新 checkout)拿不到、换装失效。
>
> ℹ️ HyperdimensionNeptunia 组等模型上游没有 `textures.cache`,加载时控制台会报该文件 **404 —— 这是无害的**(代码 try/catch 后回退到 `index.json` 默认贴图,模型照常显示)。上游没有就不补,保持一致。

## 6. 升级到上游新版(操作步骤)

上游发新版时:

1. 抓上游 `dist/` 全部产物(含 `dist/chunk/`)。
2. **新建版本目录**(用当天日期,如 `v20260720/`),把上游产物放进去。
3. 重新套用自定义:第 4 节的 asteroids 地址、布局,第 3 节的材质补丁。**路径不用改**(autoload 自动定位)。
4. 改 `templateBuild.html` 里那行的目录名 → `/live2d-widget/v20260720/autoload.js`。**这是唯一要改的引用处**,旧版本目录可留可删。
5. 若上游 `waifu-tips.json` 有新 `message.*` 字段,补进本站的(见第 4 节)。
6. 按第 7 节验证。

## 7. 缓存机制(为什么用带版本号的目录)

JS/CSS 文件名没有内容哈希,而 CDN(本站 EdgeOne)默认给 `.js` 发 `Cache-Control: max-age=31536000, immutable`。`immutable` = 浏览器**连刷新都不重新请求**。若直接覆盖同名文件,老访客浏览器一年内继续跑旧缓存,新旧混用会报错(典型:`loadlive2d is not defined`,旧 waifu-tips 撞新 autoload)。**清 CDN 边缘缓存也没用**——碰不到浏览器里的 immutable 副本。

**带版本号目录的好处**:换版本 = 换 URL,`immutable` 缓存反而正确(旧 URL 内容确实永不变)。新 HTML(`max-age=0`,每次校验)指向新目录 → 访客自动取新文件、旧缓存作废,无需清 CDN、无需手动 bump 之外的操作。模型 `live2d_api/` 不动,继续享受永久缓存,升级不重下那 136MB。

## 8. 本地验证

看板娘只在生产模板注入,`pnpm docs:dev` **不显示**。验证方式二选一:

- `pnpm docs:build` 后看 `dist/`;或
- 静态托管 `public/` 目录(如 `python -m http.server`),浏览器开一个引用 `/live2d-widget/v20260619/autoload.js` 的页面。

检查:看板娘正常显示、切换模型/换装可用、打飞机可用、控制台无 `loadlive2d`/无意外 404(Neptunia 组的 textures.cache 404 属正常)。
